### Infrastructure (AWS)
- [ ] **Terraform** — provision AWS infrastructure (VPC, subnets, security groups)
- [ ] **ECR** — container registry for Docker image (`mlops-cloud-cost-anomaly/api`)
- [ ] **S3** — bucket to store model artefacts (`best_model.joblib`, `model_metadata.json`, `feature_schema.json`)
- [ ] **EC2** — instance running the container (or ECS/EKS depending on final decision)
- [ ] **user_data script** — on instance startup: install Docker, pull ECR image, download models from S3, run container

Suggested user_data flow:
```
1. apt install docker
2. aws ecr get-login-password | docker login
3. aws s3 cp s3://bucket/models/ /home/ec2-user/models/ --recursive
4. docker run -d -p 80:8000 -v /home/ec2-user/models:/app/models:ro <ecr-uri>
```
